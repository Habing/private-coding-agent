# Slice 9 — Audit Deepening

## Context

切片 1 的 `audit.Middleware` 给每个 HTTP 请求落一行 `audit_log`，action 固定为 `http_request`。这覆盖了"访问日志"层，但有几个明显缺口：

1. **没有领域语义**——所有 action 都是 `http_request`，要追"是谁在什么时候登录失败"或"哪个用户调用了 shell.exec"必须翻 method+path+status，无法按业务事件过滤
2. **没有查询接口**——审计只能登 PG 直查 `audit_log` 表
3. **没有 RBAC**——`RoleAdmin = "admin"` 早就定义但从未被强制；任何登录用户都能调任何 endpoint
4. **没有前端展示**——管理员要看异常活动得连 DB

切片 9 收口"P0 基础审计"这一条：领域事件埋点 + admin 查询 API + Web UI 审计页。

**用户已确认本切片边界：**
1. 先写 design spec（本文件即扮演 spec + plan 双重角色）→ 再实现
2. 不做配额/速率限制（推到 P1）
3. 不做安全加固扫描（推到 P1）
4. 审计深化范围：**login + sandbox 生命周期 + session 生命周期 + WS open/close + tool 错误**——其他领域（model/memory/agent.run）保留 `http_request` middleware + 各自专表（`model_usage` / `memories`），不双写 audit_log

## Goal

- 新增 `auth.RequireAdmin()` middleware
- 新增 `GET /audit`（admin-only），支持按 `action`/`user_id`/`from`/`to`/`status`/`limit`/`offset` 过滤
- `audit.Repo` 增 `List(ctx, filter)` 方法
- 关键 handler 显式 append 领域级 audit entry（`auth.login.success` / `auth.login.failure` / `sandbox.create` / `sandbox.destroy` / `session.create` / `session.archive` / `session.ws.open` / `session.ws.close` / `tool.invoke.error`）
- Web UI 加 `/audit` 路由，仅 `user.role === 'admin'` 可见；TopBar 加 "审计" 入口（同样 admin-only）
- E2E 28 → 32 步：admin 调 `GET /audit` 看到前面步骤产生的领域 action；member 调 `GET /audit` 返 403
- mockserver / 已有迁移不动

## 非目标（明确出栈）

- 配额、rate limit、429 拦截 → P1
- 跨租户 super-admin / 全量审计 → 暂不做，所有 admin 仅看自己 tenant
- 审计导出（CSV/JSON 下载）→ 暂不做
- 审计不可篡改保护（append-only trigger、哈希链）→ P1
- `model.chat` / `memory.*` / `agent.run` 的双写 → 已在专表，不重复
- 完整 OpenTelemetry → P1

## Architecture

```
HTTP REST
  GET /audit?action=&user_id=&from=&to=&limit=&offset=    (admin only)
        |
        v
  audit.Handler.list (tenant scope from auth.FromCtx)
        |
        v
  audit.Repo.List(ctx, ListFilter) -> []Entry, total
        |
        v
  PG: SELECT ... FROM audit_log WHERE tenant_id=$1 AND ...
      ORDER BY occurred_at DESC LIMIT ... OFFSET ...

领域埋点 (现有 handler/service 显式 audit.Append):
  auth.Handler.login          -> auth.login.success|failure   target=email
  sandbox.Handler.create      -> sandbox.create               target=sandbox_id
  sandbox.Handler.destroy     -> sandbox.destroy              target=sandbox_id
  session.Service.Create      -> session.create               target=session_id
  session.Service.Archive     -> session.archive              target=session_id
  session.WSHandler.serve     -> session.ws.open / .close     target=session_id
  toolbus.Bus.Invoke          -> tool.invoke.error            target=tool_name (仅失败)

Web UI:
  TopBar -> "审计" link (admin only) -> /audit
  /audit:
    - 调 GET /audit?... 显示分页表格
    - 过滤器: action, user_id (or email), time range, status range
    - role !== 'admin' 时路由直接 redirect 到 /
```

## Action 命名规范

`<domain>.<verb>[.<outcome>]`，全小写，点分隔。固定枚举（避免拼写漂移）：

| Action | Target | Metadata 关键字段 |
|---|---|---|
| `auth.login.success` | user email | `user_id`, `tenant_slug` |
| `auth.login.failure` | user email | `reason` (bad_password / no_such_user / wrong_tenant) |
| `sandbox.create` | sandbox_id | `image` |
| `sandbox.destroy` | sandbox_id | — |
| `session.create` | session_id | `model`, `profile` |
| `session.archive` | session_id | — |
| `session.ws.open` | session_id | — |
| `session.ws.close` | session_id | `reason`, `duration_ms` |
| `tool.invoke.error` | tool_name | `error_code` |

> http_request middleware 继续写，**额外**写领域行。两层并存：access log 层 + 业务事件层。

```go
type ListFilter struct {
    TenantID   uuid.UUID  // 必填，handler 注入
    Action     string     // 前缀匹配: "auth.login" 命中 success+failure
    UserID     *uuid.UUID // 可选
    From, To   *time.Time // 可选
    MinStatus  int        // 默认 0
    MaxStatus  int        // 默认 999
    Limit      int        // 默认 50, max 200
    Offset     int        // 默认 0
}

func (r *Repo) List(ctx context.Context, f ListFilter) (entries []Entry, total int, err error)
```

`Action` 用 `LIKE $n || '%'` 前缀匹配；total 用 `COUNT(*)` 同 WHERE 跑一次。

仿 `internal/memory/repo.go` 的 `List` 方法构造 SQL（参数化、`len(args)+1` 自增占位符）。

## 关键不变量

1. **审计写盘不能阻塞业务** —— 所有 `audit.Append` 调用必须 `context.WithTimeout(context.Background(), 5s)`，失败仅 log。和现有 middleware 行为一致。
2. **租户隔离** —— `GET /audit` 永远只返 `tenant_id = cl.TenantID` 的行。没有"超级管理员看全租户"的能力（推到 P1）。
3. **两层并存** —— http_request middleware 继续保留全量 access log 行，领域埋点 **新增** 业务事件行，不替代。审计页默认按 action 前缀过滤掉 `http_request` 以减少噪声（前端默认 `?action=auth.|sandbox.|session.|tool.`，但 query 参数可显式覆盖）。
4. **Action 命名稳定** —— 一旦在 prod 使用，action 字符串成事实 API。本切片定义的枚举（见上表）以后只能加、不能删/改名。
5. **PII 最小化** —— metadata 不存 prompt 内容、不存 shell 命令原文、不存文件内容；仅存 ID、计数、错误码。`auth.login.failure` 的 target 字段记 email 是 acceptable trade-off（追失败登录必须的标识）。
6. **admin 角色硬编码** —— 本切片不引入复杂 RBAC（permission matrix、role hierarchy）；只一个 admin flag。后续如需扩展走 P1。
